Examination System – Stored Procedures Overview

1. usp_AddStudent

Purpose: Adds a new student to the system.

Description: This procedure creates a new record in the SystemUser table
with the role ‘Student’, then creates a corresponding record in the
Student table. Both operations occur inside a transaction to ensure data
consistency.

Key Features: - Uses a transaction (BEGIN TRANSACTION / COMMIT /
ROLLBACK). - Uses SCOPE_IDENTITY() to retrieve the newly created
UserID. - Ensures that both SystemUser and Student records are created
together.

Parameters: Username, PasswordHash, IntakeID, BranchID, TrackID,
FirstName, LastName, Email, Phone, DateOfBirth, NationalID.

------------------------------------------------------------------------

2. sp_AssignInstructorToCourse

Purpose: Assigns an instructor to teach a course for a specific intake,
branch, track, and academic year.

Description: The procedure checks if the same assignment already exists
to prevent duplicates. If no duplicate is found, a new record is
inserted into the InstructorCourse table.

Key Features: - Prevents duplicate assignments using IF EXISTS. - Uses
THROW to raise an error if the assignment already exists.

Parameters: InstructorID, CourseID, IntakeID, BranchID, TrackID,
AcademicYear.

------------------------------------------------------------------------

3. sp_SearchStudents

Purpose: Searches for students based on optional filters.

Description: Returns student information from the view
vw_StudentFullInfo. The procedure allows filtering by Intake, Branch,
Track, or Student Name.

Key Features: - Flexible filtering using optional parameters. - Uses
LIKE for partial name searching. - Returns complete student information
from a view.

Parameters: IntakeID, BranchID, TrackID, SearchName.

------------------------------------------------------------------------

4. sp_AddQuestion

Purpose: Adds a new question to the question pool.

Description: This procedure inserts a question created by an instructor
into the Question table. Questions can be of different types such as
MCQ, True/False, or Text.

Key Features: - Stores course ID and instructor ownership. - Supports
difficulty levels. - Returns the ID of the newly inserted question.

Parameters: CourseID, InstructorID, QuestionType, QuestionText,
ModelAnswer, DifficultyLevel.

------------------------------------------------------------------------

5. sp_AddQuestionChoice

Purpose: Adds answer choices for MCQ or True/False questions.

Description: This procedure inserts a choice into the QuestionChoice
table for a specific question. It validates that the question type
supports choices and enforces a maximum of two choices for True/False
questions.

Key Features: - Validates question type. - Restricts True/False
questions to two choices. - Stores the correct answer flag.

Parameters: QuestionID, ChoiceText, IsCorrect, ChoiceOrder.

------------------------------------------------------------------------

6. sp_UpdateQuestion

Purpose: Updates an existing question.

Description: Allows the instructor who created the question to modify
its text, model answer, difficulty level, or active status.

Key Features: - Ensures that only the question owner can update it. -
Uses ISNULL to allow partial updates.

Parameters: QuestionID, InstructorID, QuestionText, ModelAnswer,
DifficultyLevel, IsActive.

------------------------------------------------------------------------

7. sp_DeleteQuestion

Purpose: Deletes a question from the system.

Description: Checks whether the question belongs to the instructor and
ensures it is not used in an exam before allowing deletion.

Key Features: - Prevents deletion of questions used in exams. - Ensures
instructor ownership before deletion.

Parameters: QuestionID, InstructorID.

------------------------------------------------------------------------

8. sp_SearchQuestions

Purpose: Searches the question pool.

Description: Retrieves questions from the vw_QuestionPool view based on
optional filters such as course, question type, difficulty level, or
text content.

Key Features: - Flexible filtering parameters. - Uses LIKE for keyword
search. - Only returns active questions.

Parameters: CourseID, QuestionType, Difficulty, SearchText.
